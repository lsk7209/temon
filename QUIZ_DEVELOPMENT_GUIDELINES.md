# 퀴즈 개발 가이드라인 및 개선점

## 📋 목적
추가 개발되는 모든 퀴즈가 사용자에게 즐겁고 의미 있는 경험을 제공하도록 보장하기 위한 가이드라인

---

## 🚨 절대 금지 사항 (발견 시 즉시 수정)

### 1. 템플릿 질문 패턴 (절대 사용 금지)

#### ❌ 추상적이고 단순한 질문 (즉시 삭제)
```
BAD: "식사 빈도는?"
BAD: "요리 계획은?"
BAD: "음식 선택 기준은?"
BAD: "식사 패턴은?"
BAD: "음식 공유 빈도는?"
BAD: "화장실 사용 감정은?"
BAD: "화장실 사용 이유는?"
BAD: "화장실 사용 후기는?"
```

#### ❌ 반복되는 템플릿 질문 (절대 사용 금지)
```
BAD: "{주제} 사용 감정은?"
BAD: "{주제} 사용 이유는?"
BAD: "{주제} 사용 후기는?"
BAD: "{주제} 선택 방식은?"
BAD: "{주제} 선택 기준은?"
BAD: "{주제} 변경은?"
BAD: "{주제} 참여는?"
BAD: "{주제} 패턴은?"
BAD: "{주제} 빈도는?"
BAD: "{주제} 계획은?"
BAD: "{주제} 준비는?"
BAD: "{주제} 경험 활용은?"
BAD: "{주제} 후 행동은?"
BAD: "{주제} 유연성은?"
BAD: "{주제} 대상은?"
BAD: "{주제} 스타일은?"
```

**⚠️ 이런 질문들은 재미없고 단조로우며 바이럴 될 수 없습니다!**
**⚠️ 발견 즉시 삭제하고 재작성해야 합니다!**

#### ❌ 모호한 답변 선택지
```
BAD: "감각과 직감" vs "논리와 정보"
BAD: "즉시 선택" vs "신중하게 선택"
BAD: "새로운 시도" vs "익숙한 방식"
BAD: "다른 사람과 함께" vs "혼자만"
```

## ✅ 필수 사항 (반드시 지켜야 할 것)

### 🎯 테몬 퀴즈의 핵심 철학
**"사람들이 공유하고 싶어하는 재미있는 퀴즈"**
- 재미있고 유쾌해야 함
- 바이럴 될 수 있어야 함
- 공유하고 싶게 만들어야 함
- 웃음과 공감을 유발해야 함

### 1. 질문 설계 원칙 (바이럴 & 재미있게)

#### ✅ 구체적이고 재미있는 상황 묘사
```
GOOD: "화장실에서 핸드폰을 떨어뜨렸을 때"
GOOD: "화장실 문이 잠겨있는데 급할 때"
GOOD: "화장실에서 이상한 소리가 들릴 때"
GOOD: "요리 중 불이 꺼졌는데 재료는 다 넣었을 때"
GOOD: "요리 중 타이머가 울렸는데 아직 안 익었을 때"
GOOD: "배달 음식이 1시간 넘게 안 올 때"
GOOD: "배달 음식이 왔는데 주문한 게 아닐 때"
```

#### ✅ 공감대를 형성하는 일상적 상황
```
GOOD: "맛있는 음식을 먹다가 실수로 떨어뜨렸을 때"
GOOD: "맛있는 음식을 먹다가 배가 터질 것 같을 때"
GOOD: "맛있는 음식을 먹다가 친구가 와서 나눠야 할 때"
GOOD: "요리 중 재료를 깜빡 잊고 안 넣었을 때"
GOOD: "요리 중 친구가 와서 맛보자고 할 때"
```

#### ✅ 웃음과 재미를 유발하는 상황
```
GOOD: "요리한 음식이 예쁘지 않게 나왔을 때"
GOOD: "요리한 음식을 사진 찍으려는데 맛이 없을 때"
GOOD: "요리한 음식을 친구에게 보여줬는데 반응이 없을 때"
GOOD: "요리 후 주방이 전쟁터가 되었을 때"
GOOD: "요리 후 설거지가 산더미처럼 쌓였을 때"
```

#### ✅ 공유하고 싶게 만드는 재미있는 상황
```
GOOD: "새로운 매장을 발견했을 때"
GOOD: "웨이터가 친절하게 맞이해줄 때"
GOOD: "예약한 식당이 예상과 다를 때"
GOOD: "메뉴 가격이 예상보다 비쌀 때"
GOOD: "할인 쿠폰이 있는 식당을 선택할 때"
```

### 2. 답변 선택지 설계 원칙

#### ✅ 구체적인 행동과 생각을 묘사
```
GOOD: "즉시 들어가서 둘러본다"
GOOD: "리뷰를 확인하고 신중하게 결정한다"
GOOD: "가격 대비 품질을 따져본다"
GOOD: "분위기와 느낌으로 선택한다"
```

#### ✅ MBTI 태그 분배 균형
- 각 질문은 2개의 답변 선택지 제공
- 각 답변은 1-2개의 MBTI 태그 포함
- 12개 질문으로 4개 차원(E/I, S/N, T/F, J/P) 모두 커버
- 각 차원별로 최소 3개 이상의 질문에서 측정

#### ✅ 답변 선택지 예시
```typescript
{
  id: 1,
  q: "새로운 매장을 발견했을 때",
  a1: { text: "즉시 들어가서 둘러본다", tags: ["E", "P"] },
  a2: { text: "리뷰를 확인하고 신중하게 결정한다", tags: ["I", "J"] },
},
{
  id: 2,
  q: "매장 선택 기준",
  a1: { text: "가격, 위치, 평점 등 객관적 정보를 비교한다", tags: ["T", "S"] },
  a2: { text: "분위기와 느낌으로 선택한다", tags: ["F", "N"] },
},
```

### 4. 퀴즈 페이지 UX/UI 개선

#### ✅ 사용자 경험 개선 사항

1. **진행 상황 표시**
   - 현재 질문 번호 / 전체 질문 수
   - 진행률 프로그레스 바 (시각적 피드백)
   - 색상 그라데이션으로 시각적 즐거움 제공

2. **질문 표시**
   - 큰 폰트로 명확하게 표시
   - 읽기 쉬운 간격과 레이아웃
   - 다크 모드 지원

3. **답변 선택**
   - 큰 버튼으로 클릭하기 쉽게
   - 호버 효과와 선택 피드백
   - 선택 후 자동 진행 (0.5초 딜레이)

4. **로딩 및 피드백**
   - "결과 분석 중..." 메시지
   - 부드러운 전환 애니메이션
   - 중복 클릭 방지

5. **접근성**
   - 키보드 네비게이션 지원
   - ARIA 레이블 및 역할 속성
   - 스크린 리더 지원

#### ✅ 기능 개선 사항

1. **이전 질문으로 돌아가기**
   - 답변 수정 가능
   - 진행 상황 저장

2. **공유 기능**
   - 결과 페이지에서 SNS 공유 버튼
   - 이미지 생성 및 다운로드

3. **결과 저장**
   - 로컬 스토리지에 결과 저장
   - 이전 결과 확인 가능

4. **애니메이션**
   - 질문 전환 시 부드러운 애니메이션
   - 결과 페이지 등장 효과

## 📝 개발 체크리스트

### 질문 설계 체크리스트 (필수 확인)
- [ ] **재미있는가?** - 읽고 웃음이 나오는가?
- [ ] **구체적인가?** - 상황이 명확하게 그려지는가?
- [ ] **공감되는가?** - "아, 나도 그런 적 있어!"라고 생각되는가?
- [ ] **공유하고 싶은가?** - 친구에게 보여주고 싶은가?
- [ ] **바이럴 될 수 있는가?** - SNS에 올리고 싶은가?
- [ ] **템플릿 패턴을 사용하지 않았는가?** - "{주제} 사용 감정은?" 같은 패턴이 없는가?
- [ ] **일상적이고 친근한가?** - 누구나 경험할 수 있는 상황인가?
- [ ] **유쾌하고 긍정적인가?** - 부정적이거나 무거운 내용이 아닌가?
- [ ] 모든 질문이 구체적인 시나리오를 묘사하는가?
- [ ] 질문이 추상적이거나 모호하지 않은가?
- [ ] 각 질문이 퀴즈 테마와 직접적으로 관련이 있는가?
- [ ] 답변 선택지가 구체적인 행동이나 생각을 묘사하는가?
- [ ] MBTI 태그 분배가 균형 잡혀 있는가? (각 차원 최소 3개 질문)

### 결과 페이지 체크리스트
- [ ] summary가 매력적이고 기억에 남는가?
- [ ] description이 200-300자로 충분히 상세한가?
- [ ] traits가 5개 이상이고 구체적인가?
- [ ] picks가 3개 이상이고 실용적인가?
- [ ] tips가 실행 가능한 조언인가?
- [ ] compatibility가 3개 타입 이상 포함하는가?

### UX/UI 체크리스트
- [ ] 진행률 표시가 명확한가?
- [ ] 질문이 읽기 쉬운가?
- [ ] 답변 버튼이 클릭하기 쉬운가?
- [ ] 애니메이션이 부드러운가?
- [ ] 접근성이 고려되었는가?
- [ ] 다크 모드가 지원되는가?

### 기능 체크리스트
- [ ] 이전 질문으로 돌아가기가 작동하는가?
- [ ] 공유 기능이 작동하는가?
- [ ] 결과 저장이 작동하는가?
- [ ] 로딩 상태가 표시되는가?

## 🔥 바이럴 질문 작성 마법의 공식

**"일상 + 예상치 못한 상황 + 재미 = 바이럴 질문"**

예시:
