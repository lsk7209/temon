# 퀴즈 개발 템플릿

## 퀴즈 기본 정보
- **ID**: `quiz-id` (kebab-case)
- **제목**: "퀴즈 제목"
- **설명**: "퀴즈에 대한 간단한 설명"
- **카테고리**: "카테고리명"
- **태그**: ["태그1", "태그2", "태그3"]

## 질문 설계 (12개)

### 🚨 질문 작성 필수 규칙

#### ❌ 절대 사용 금지 패턴
```
❌ "{주제} {행동}은?"
❌ "{주제} {행동}는?"
❌ "{주제} {행동}이?"
❌ "{주제} {행동}가?"
❌ "연락은?", "톤은?", "기념일은?", "정산은?", "관리는?", "처리는?"
```

**⚠️ "은?", "는?", "이?", "가?"로 끝나는 질문은 모두 템플릿 패턴입니다!**

#### ✅ 필수 사용 패턴
```
✅ "{상황}에서 {행동}할 때"
✅ "{상황}에서 {행동}했을 때"
✅ "{상황} 중에 {행동}할 때"
✅ "{상황} 전에 {행동}할 때"
✅ "{상황} 후에 {행동}할 때"
```

**질문 길이:**
- 질문은 짧고 간결하게 작성 (20자 이내 권장)
- 핵심 상황만 간단명료하게 표현

### 질문 작성 가이드
- ✅ 구체적인 시나리오 묘사 (필수!)
- ✅ 구체적인 시점 포함 ("~할 때", "~했을 때" 등)
- ✅ 실제 상황 기반
- ✅ 행동과 감정 연결
- ✅ 짧고 간결하게 (20자 이내)
- ❌ 추상적이거나 모호한 질문 금지
- ❌ 템플릿 패턴 사용 금지 ("은?", "는?", "이?", "가?")

### ❌ 나쁜 질문 예시 (절대 사용 금지)
```typescript
const questions = [
  {
    id: 1,
    q: "치킨 선택은?",  // ❌ 템플릿 패턴!
    a1: { text: "항상 같은 메뉴", tags: ["J"] },
    a2: { text: "다양한 메뉴", tags: ["P"] },
  },
  {
    id: 2,
    q: "편의점에서 행동은?",  // ❌ 템플릿 패턴!
    a1: { text: "빠르게 쇼핑", tags: ["J"] },
    a2: { text: "구경하며 쇼핑", tags: ["P"] },
  },
  // 이런 질문들은 즉시 재작성해야 합니다!
]
```

### MBTI 태그 분배 체크
- E/I: 최소 3개 질문
- S/N: 최소 3개 질문
- T/F: 최소 3개 질문
- J/P: 최소 3개 질문

## 개발 체크리스트

### 질문 설계 (필수 체크!)

**1단계: 템플릿 패턴 검사 (가장 중요!)**
- [ ] 질문이 "은?", "는?", "이?", "가?"로 끝나지 않는가?
- [ ] 질문이 구체적인 시점("~할 때", "~했을 때", "~일 때")을 포함하는가?
- [ ] 질문이 20자 이내로 짧고 간결한가?

**2단계: 질문 품질 검사**
- [ ] 재미있고 유쾌한가?
- [ ] 구체적인 상황을 묘사하는가?
- [ ] 공감대를 형성하는가?
- [ ] 공유하고 싶게 만드는가?

**3단계: 기술적 검사**
- [ ] 모든 질문이 구체적인 시나리오인가?
- [ ] MBTI 태그 분배가 균형 잡혀 있는가? (각 차원 최소 3개)
- [ ] 답변 선택지가 구체적인 행동을 묘사하는가?

**⚠️ 1단계 검사를 통과하지 못하면 즉시 재작성해야 합니다!**

### 결과 페이지
- [ ] summary가 매력적인가?
- [ ] description이 충분히 상세한가? (200-300자)
- [ ] traits가 5개 이상인가?
- [ ] picks가 3개 이상인가?
- [ ] tips가 실행 가능한가?

### UX/UI
- [ ] 진행률 표시가 명확한가?
- [ ] 애니메이션이 부드러운가?
- [ ] 접근성이 고려되었는가?

### 기능
- [ ] 이전 질문으로 돌아가기가 작동하는가?
- [ ] 공유 기능이 작동하는가?
- [ ] 결과 저장이 작동하는가?
