# 퀴즈 개발 템플릿

## 퀴즈 기본 정보
- **ID**: `quiz-id` (kebab-case)
- **제목**: "퀴즈 제목"
- **설명**: "퀴즈에 대한 간단한 설명"
- **카테고리**: "카테고리명"
- **태그**: ["태그1", "태그2", "태그3"]

## 질문 설계 (12개)

### 질문 작성 가이드
- ✅ 구체적인 시나리오 묘사
- ✅ 실제 상황 기반
- ✅ 행동과 감정 연결
- ❌ 추상적이거나 모호한 질문 금지
- ❌ 템플릿 패턴 사용 금지

### 질문 예시
```typescript
const questions = [
  {
    id: 1,
    q: "구체적인 시나리오를 묘사하는 질문",
    a1: { text: "구체적인 행동이나 생각을 묘사하는 답변", tags: ["MBTI1", "MBTI2"] },
    a2: { text: "구체적인 행동이나 생각을 묘사하는 답변", tags: ["MBTI3", "MBTI4"] },
  },
  // ... 총 12개
]
```

### MBTI 태그 분배 체크
- E/I: 최소 3개 질문
- S/N: 최소 3개 질문
- T/F: 최소 3개 질문
- J/P: 최소 3개 질문

## 파일 구조

```
app/tests/{quiz-id}/
├── page.tsx                    # 인트로 페이지
├── test/
│   ├── page.tsx               # 퀴즈 질문 페이지
│   └── result/
│       └── page.tsx           # 결과 페이지
```

## 개발 체크리스트

### 질문 설계
- [ ] 모든 질문이 구체적인 시나리오인가?
- [ ] 템플릿 패턴을 사용하지 않았는가?
- [ ] MBTI 태그 분배가 균형 잡혀 있는가?

### 결과 페이지
- [ ] summary가 매력적인가?
- [ ] description이 충분히 상세한가? (200-300자)
- [ ] traits가 5개 이상인가?
- [ ] picks가 3개 이상인가?
- [ ] tips가 실행 가능한가?

### UX/UI
- [ ] 진행률 표시가 명확한가?
- [ ] 애니메이션이 부드러운가?
- [ ] 접근성이 고려되었는가?

### 기능
- [ ] 이전 질문으로 돌아가기가 작동하는가?
- [ ] 공유 기능이 작동하는가?
- [ ] 결과 저장이 작동하는가?
