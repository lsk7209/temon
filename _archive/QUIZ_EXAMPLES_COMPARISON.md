# 퀴즈 질문 및 결과 페이지 예시 비교

## 📊 질문 예시 비교

### ❌ 나쁜 예시 (피해야 할 것)

#### 예시 1: 추상적이고 모호한 질문
```typescript
const questions = [
  {
    id: 1,
    q: "식사 빈도는?",
    a1: { text: "규칙적으로 자주 먹기", tags: ["E"] },
    a2: { text: "가끔 불규칙하게 먹기", tags: ["I"] },
  },
  {
    id: 2,
    q: "식사 계획은?",
    a1: { text: "미리 계획", tags: ["J"] },
    a2: { text: "그때그때 선택", tags: ["P"] },
  },
  {
    id: 3,
    q: "식사 선택 기준은?",
    a1: { text: "감각과 직감", tags: ["F"] },
    a2: { text: "논리와 정보", tags: ["T"] },
  },
]
```

**문제점:**
- 질문이 너무 추상적이고 모호함
- 답변이 구체적이지 않아 사용자가 선택하기 어려움
- 템플릿 패턴을 그대로 사용
- 퀴즈 테마와의 연결이 약함

### ✅ 좋은 예시 (따라야 할 것)

#### 예시 1: 구체적이고 시나리오 기반 질문
```typescript
const questions = [
  {
    id: 1,
    q: "웨이터가 친절하게 맞이해줄 때",
    a1: { text: "기분이 좋아지고 만족감이 높아진다", tags: ["F"] },
    a2: { text: "당연한 서비스로 생각하고 크게 신경 쓰지 않는다", tags: ["T"] },
  },
  {
    id: 2,
    q: "서비스 속도가 빠를 때",
    a1: { text: "효율적이라고 생각하고 좋아한다", tags: ["T", "J"] },
    a2: { text: "여유롭게 즐기고 싶어서 아쉽다", tags: ["F", "P"] },
  },
  {
    id: 3,
    q: "추가 서비스(물, 수저, 냅킨 등)를 먼저 제공할 때",
    a1: { text: "신경 쓰는 서비스라고 느껴서 좋다", tags: ["F", "S"] },
    a2: { text: "필요할 때 요청하는 게 더 효율적이다", tags: ["T", "N"] },
  },
]
```

**장점:**
- 구체적인 시나리오를 묘사
- 실제 상황을 상상할 수 있음
- 답변이 구체적인 감정과 행동을 묘사
- 퀴즈 테마(서비스 선호도)와 직접적으로 연결

## 📄 결과 페이지 예시 비교

### ❌ 나쁜 예시 (피해야 할 것)

```typescript
{
  "ISTJ": {
    summary: "계획적인 사람",
    description: "계획을 세우는 것을 좋아합니다.",
    traits: [
      "계획적",
      "체계적"
    ],
    picks: [
      "계획적인 것"
    ],
    tips: [
      "더 계획적으로"
    ],
    compatibility: {
      best: ["ISTJ"],
      good: [],
      avoid: []
    }
  }
}
```

**문제점:**
- summary가 너무 짧고 단순함
- description이 불충분함 (20자 미만)
- traits가 2개뿐이고 구체적이지 않음
- picks가 모호하고 실용적이지 않음
- tips가 실행 불가능함
- compatibility 정보가 부족함

---

- tips가 실행 가능하고 구체적임
- compatibility 정보가 충분함

## 🎯 질문 작성 가이드

### 단계별 질문 작성 프로세스

#### 1단계: 테마 분석
- 퀴즈의 핵심 테마 파악
- 사용자가 경험할 수 있는 실제 상황 생각하기

#### 2단계: 시나리오 구상
- 구체적인 상황 설정
- 감정과 행동이 드러나는 순간 포착

#### 3단계: 질문 작성
- "~할 때", "~했을 때" 같은 구체적 시점 사용
- 사용자가 쉽게 상상할 수 있는 상황 묘사

#### 4단계: 답변 작성
- 구체적인 행동이나 감정 묘사
- 모호한 표현 피하기

#### 5단계: MBTI 태그 분배
- 각 차원(E/I, S/N, T/F, J/P) 최소 3개 질문에서 측정
- 균형 잡힌 분배 확인

## 📝 결과 페이지 작성 가이드

### 단계별 결과 페이지 작성 프로세스

#### 1단계: Summary 작성
- 한 줄로 요약하되 매력적으로
- 사용자가 공유하고 싶게 만들기

#### 2단계: Description 작성
- 200-300자로 상세히 설명
- 사용자의 특징을 구체적으로 묘사
- 긍정적이면서도 현실적으로

#### 3단계: Traits 작성
- 5개 이상의 구체적 특징
- 행동과 생각을 구체적으로 묘사
- 긍정적 특징 위주로

#### 4단계: Picks 작성
- 3개 이상의 실용적 추천
- 구체적인 예시 포함
- 사용자가 실제로 활용할 수 있게

#### 5단계: Tips 작성
- 실행 가능한 개선 팁
- 구체적인 행동 제안
- 긍정적이고 건설적으로

#### 6단계: Compatibility 작성
- best: 2-3개 타입
- good: 2-3개 타입
- avoid: 1-2개 타입
- 이유를 간단히 설명할 수 있게
