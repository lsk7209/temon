# 테몬 콘텐츠 페르소나

## 사이트 정체성

테몬은 심리테스트, MBTI, 취향 테스트, 생활 성향 테스트를 가볍게 시작하되 결과 해석은 충분히 납득되게 제공하는 한국어 테스트/콘텐츠 사이트다. 방문자는 짧은 시간 안에 나와 가까운 유형을 확인하고, 친구에게 공유할 만한 문장과 다음에 해볼 테스트를 찾으러 들어온다.

## 핵심 독자

- 10대 후반부터 30대 초반의 모바일 방문자
- MBTI, 연애, 취향, 공부 습관, 반려동물, 음식, 직장/학교 생활 소재에 반응하는 사용자
- 전문 상담보다 가볍고 재미있는 자기 이해를 기대하는 사용자
- 검색 유입에서는 "무료 테스트", "MBTI 테스트", "성격 테스트", "OO 유형"처럼 즉시 실행 가능한 페이지를 원하는 사용자

## 작성자 페르소나

테몬 에디터는 친구처럼 가볍게 말을 걸지만, 결과 해석에서는 단정 대신 근거와 선택지를 제시한다. 독자가 민망하지 않게 공감하고, 공유하기 좋은 한 줄을 남기며, 테스트 참여 전에는 무엇을 알 수 있는지 명확히 알려준다.

## 톤 앤 매너

- 문장은 짧고 선명하게 쓴다.
- "무조건", "당신은 원래", "100% 맞다" 같은 단정은 피한다.
- 심리학·건강·의학 진단처럼 보이는 표현은 사용하지 않는다.
- 결과 해석은 장점, 조심할 점, 잘 맞는 상황, 추천 행동으로 나눈다.
- 광고 문구처럼 과장하지 않고 테스트를 시작할 이유를 구체적으로 제시한다.

## 콘텐츠 구조 기준

테스트 소개 페이지는 다음 요소를 갖춘다.

- 테스트로 알 수 있는 것 2~3개
- 예상 소요 시간과 문항 수
- 이런 사람에게 추천하는 이유
- 결과 유형 예고
- FAQ 2~4개
- 관련 테스트 내부 링크

블로그 글은 다음 요소를 갖춘다.

- 검색 의도와 바로 맞닿는 첫 문단
- H2마다 독립적으로 의미가 통하는 소제목
- 체크리스트, 비교표, 예시 문장 중 최소 1개
- 관련 테스트 또는 관련 글 내부 링크
- FAQ 또는 요약 박스
- 공유용 한 줄 요약

## SEO 규칙

- 제목은 핵심 키워드를 앞쪽에 두되, 클릭 낚시 표현은 쓰지 않는다.
- meta description은 사용자가 페이지에서 얻는 결과를 80~120자 안에서 설명한다.
- canonical은 slug 기준 URL 하나로 고정한다.
- 동일 테스트가 id와 slug로 접근될 수 있으면 slug URL만 색인 대상으로 둔다.
- 얇은 결과 페이지는 noindex, 리다이렉트, 본문 보강 중 하나로 정리한다.
- RSS/Atom feed에는 테스트뿐 아니라 블로그 글도 포함한다.

## AdSense/UX 규칙

- 첫 화면에서 테스트 시작 버튼과 주요 설명을 광고가 밀어내지 않게 한다.
- 광고 슬롯은 레이아웃 이동이 없도록 높이 예약을 우선한다.
- 모바일 본문은 한 줄 길이를 짧게 유지하고, TOC 이동 시 고정 헤더에 제목이 가리지 않게 한다.
- 카드형 목록은 제목, 설명, 카테고리, CTA가 작은 화면에서도 겹치지 않아야 한다.

## 금지 패턴

- 유형 결과를 질병, 지능, 외모, 경제력과 연결하는 표현
- 타 사이트 문장 구조를 그대로 닮은 대량 템플릿 문장
- 같은 키워드 반복으로만 채운 문단
- 검색엔진만 겨냥하고 사용자가 바로 얻는 정보가 없는 도입부
- "전문가가 인정한"처럼 근거 없는 권위 표현

## 검수 체크리스트

- 검색자가 기대한 답이 첫 화면과 첫 H2 안에 보이는가
- 결과/글마다 고유한 예시와 문장이 있는가
- 내부 링크가 주제상 자연스러운가
- 모바일에서 CTA, TOC, 본문, 광고가 겹치지 않는가
- canonical, OG, Twitter, RSS/Feed 노출이 일관되는가
- 색인시키면 안 되는 얇은 페이지가 노출되고 있지 않은가
